A history of lame

The LameStation is a game console designed from the ground up with learning in mind. It had to be, since it started as a school project.

That's right! The LameStation is five years in the making, and Brett designed the first prototype by himself while he was still a student at UC Irvine.

That's a big part of why it looks the way it does today:

  • Brett wasn't good at soldering, so the LameStation uses all through-hole components, large solder pads, and can be tested at multiple points during assembly.
  • Brett had to finish someday, so a complicated multi-chip system with nine million features was not going to work. The LameStation is a system complicated enough to make it work, but nothing more.
  • Brett wanted to learn something, so using off-the-shelf, working pieces to build it would have defeated the whole purpose. Every piece had to be created from scratch and had to be easy to understand.

So why is it lame?

Because it's the best that 1989 had to offer!

But really, it's all the things it doesn't offer that makes it special.

The LameStation does not...

...have much memory

With 32kB of memory, the entire platform, every line of code that goes into your game, can be read through in a day (though you may not understand all of it!)

...support color

Color makes everything bigger: more memory, more code, more speed. The LameStation's tiny, black and white screen is easy to understand and even easier to write a driver for.

...have an operating system

Whenever you write a LameStation game, you're programming the hardware directly, so you can see exactly what goes into making your game work.
